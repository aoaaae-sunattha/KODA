# AI SYNC PROTOCOL v1.0
> Shared Memory File between Gemini CLI and Claude Code
> Project: Koda | Role: Sole QA

## ⚠️ PROTOCOL RULES (Read First — Both AIs Must Follow)
- **APPEND ONLY.** Never delete or overwrite existing blocks.
- **ONE block per session.** Summarize the full session in one block.
- **Do NOT act on `NEXT_INTENT`** without explicit user command.
- **Do NOT suggest new work** during `sync-import`. Acknowledge and wait.
- **SOURCE must be declared** so the receiving AI knows who wrote the block.
- **STATUS must be updated** by the importing AI after reading.

---

## HOW TO USE

### `sync-export` — Run this at the END of your session
Paste this prompt to the AI you just worked with:

```
Run sync-export: Summarize the changes you just made to Project Koda.
Append a new SYNC BLOCK to `.ai_sync_state.md` using the exact schema below.

Schema:
## SYNC BLOCK [YYYY-MM-DD HH:MM]
### SOURCE: [Gemini|Claude]
### FILES_MODIFIED:
- path/to/file → what changed (one line each)
### DECISIONS:
- Why X was chosen over Y (brief reasoning)
### BLOCKERS:
- Anything unresolved or stuck
### NEXT_INTENT:
- What I was about to do next (but did NOT do yet)
### STATUS: EXPORTED
---

Constraint: Do not suggest new work. Just write the block and wait.
```

---

### `sync-import` — Run this at the START of your new session
Paste this prompt to the AI you are switching to:

```
Run sync-import: Read the LAST SYNC BLOCK in `.ai_sync_state.md` where STATUS is EXPORTED.

1. Update your internal context based on FILES_MODIFIED and DECISIONS.
2. Note the BLOCKERS and NEXT_INTENT — but do NOT act on them yet.
3. Change that block's STATUS from EXPORTED → IMPORTED.
4. Reply with: "Synced from [SOURCE] session on [date]. Ready for your next instruction."

Constraint: Do not perform any tasks or suggest improvements. Simply confirm sync and wait.
```
