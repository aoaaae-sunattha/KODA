# CI/CD Pipeline for E2E Testing — Design Spec

**Date:** 2026-04-01
**Status:** Approved
**Audience:** Interview showcase + real development workflow

---

## Overview

Add a GitHub Actions CI pipeline that runs on every PR to `main`. The pipeline validates code quality (lint + types), runs unit tests (Vitest), and runs E2E tests (Playwright). Test results are surfaced three ways: GitHub Artifacts, PR comments, and committed report files.

## Pipeline Architecture

```
Open/Update PR to main
       │
       ├──► Job 1: lint-typecheck        ~20s
       │       └─ npm run lint → tsc -b
       │
       ├──► Job 2: unit-tests            ~30s
       │       └─ npm run test (Vitest)
       │
       └──► Job 3: e2e-tests             ~2min
               ├─ Install Playwright browsers (cached)
               ├─ Start dev server (via playwright.config.ts webServer)
               ├─ npx playwright test
               ├─ Upload HTML report → GitHub Artifact (7-day retention)
               ├─ Post pass/fail summary as PR comment
               └─ Commit markdown report to QA/REPORTS/
```

All 3 jobs run in parallel. Each job is independent — a lint failure does not block E2E results.

## CI Trigger

```yaml
on:
  pull_request:
    branches: [main]
```

Does NOT run on push to feature branches. Only on PR open/update/reopen to main.

## Job Details

### Job 1: lint-typecheck

- **Runner:** ubuntu-latest
- **Node:** 22 (matches project nvm requirement)
- **Steps:** `npm ci` → `npm run lint` → `npx tsc -b`
- **Purpose:** Catch syntax errors, type errors, lint violations

### Job 2: unit-tests

- **Runner:** ubuntu-latest
- **Node:** 22
- **Steps:** `npm ci` → `npm run test`
- **Purpose:** Run all 6 Vitest test files (CheckoutFlow, RefundEngine, RiskGuard, CardManagement, MerchantPortal, QA_Simulation)

### Job 3: e2e-tests

- **Runner:** ubuntu-latest
- **Node:** 22
- **Steps:**
  1. `npm ci` (in `app/`)
  2. `npx playwright install --with-deps chromium` (cached via `actions/cache`)
  3. `npx playwright test` (webServer config starts dev server automatically)
  4. **Always (pass or fail):**
     - Upload `playwright-report/` as GitHub Artifact (7-day retention)
     - Post/update PR comment with pass/fail summary table
     - Generate markdown report and commit to `QA/REPORTS/YYYY-MM-DD/`

#### PR Comment Format

Uses `actions/github-script` to post or update a single comment (no spam):

```markdown
## E2E Test Results

| Suite | Tests | Passed | Failed | Duration |
|-------|-------|--------|--------|----------|
| Login | 8     | 8      | 0      | 12s      |

**Status:** All tests passed
**Report:** [Download full report](link-to-artifact)
```

#### Report Commit

- File: `QA/REPORTS/YYYY-MM-DD/e2e-<feature>-results.md`
- Also overwrites `QA/REPORTS/LATEST_REPORT.md`
- Commit message: `ci: update E2E report [skip ci]` (prevents re-triggering the pipeline)
- Uses `github-actions[bot]` as committer

## Report File Naming

Fits existing `QA/REPORTS/` structure:

```
QA/REPORTS/
  2026-04-01/
    e2e-login-results.md        ← generated by CI
    v0.0.0-login.md             ← existing manual report
  LATEST_REPORT.md              ← overwritten each CI run
```

## Branching Strategy

Fresh branch per feature PR, branched from latest `main`:

| Branch | PR Title | Key Files |
|--------|----------|-----------|
| `e2e/login` | E2E: Login flow automation | `.github/workflows/ci.yml`, `QA/README.md` update |
| `e2e/dashboard` | E2E: Dashboard automation | `playwright/tests/dashboard.spec.ts`, `playwright/pages/DashboardPage.ts` |
| `e2e/store-checkout` | E2E: Store & Checkout | `playwright/tests/store-checkout.spec.ts`, `playwright/pages/StorePage.ts` |
| `e2e/refunds` | E2E: Refunds | `playwright/tests/refunds.spec.ts` |
| `e2e/risk-kyc` | E2E: Risk & KYC | `playwright/tests/risk-kyc.spec.ts` |
| `e2e/card-management` | E2E: Card Management | `playwright/tests/cards.spec.ts`, `playwright/pages/CardsPage.ts` |
| `e2e/merchant` | E2E: Merchant Portal | `playwright/tests/merchant.spec.ts`, `playwright/pages/MerchantPage.ts` |

PR #1 (`e2e/login`) includes the CI workflow file. Subsequent PRs only add spec + page object files.

**Modified files:**
- `QA/README.md` — add Playwright + POM + CI section

**Already exists (no changes):**

- `playwright/pages/LoginPage.ts` — Page Object Model

## Technical Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| CI trigger | PR to main only | Saves GitHub Actions minutes, runs when it matters |
| Jobs | 3 parallel | Independent concerns, faster feedback, no blocking |
| Node version | 22 | Matches project nvm requirement (system node 18 has icu4c mismatch) |
| Browser | Chromium only | Matches `playwright.config.ts`, sufficient for mockup |
| Browser caching | `actions/cache` on `~/.cache/ms-playwright` | Saves ~40s per run |
| PR comment | `github-script` (update existing) | No comment spam on repeated pushes |
| Report commit | `[skip ci]` in message | Prevents infinite CI loop |
| Artifact retention | 7 days | GitHub default, sufficient for PR review cycle |
| Working directory | `app/` for lint, build, unit tests; repo root for `npx playwright test` | `playwright.config.ts` is at root, `package.json` is in `app/` |

## Out of Scope

- Deployment pipeline (no production deploy for mockup)
- Multiple browser testing (Chromium only)
- Scheduled/cron test runs
- Slack/Discord notifications
- Code coverage reporting
